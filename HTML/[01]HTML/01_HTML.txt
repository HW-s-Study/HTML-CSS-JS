### 텍스트 입력하기 (태그 알아보기)
- <hn> - </hn> : heading(제목), h1 ~ h6까지 제목 텍스트를 크기별로 표시 가능 (숫자가 커질수록 작아짐) <br />
- <p> - </p> : paragraph(단락), 텍스트 앞뒤로 빈 줄이 생기면서 단락을 생성 (웹 브라우저가 줄바꿈을 인식하지 못하므로 단락을 구분) <br />
- <br> : A forced line-break(강제 줄바꿈), 웹 브라우저가 인식할 수 있도록 줄바꿈 (하지만 여러 요인으로 남발하지는 않고 단락을 구분할 때에는 p 태그를 사용) <br />
- <blockquote> - </blockquote> : 다른 출처에서 인용된 인용 섹션을 나타내는 독립적인 인용 구간 표시 태그 <br />

#### 태그의 중첩과 목록
<li> 태그만 사용해도 아무런 문제가 되지 않지만, 상위 목록과 하위 목록에 대한 구분이 필요할 때가 존재하기에 태그를 중첩해 사용한다 <br />
why ? 브라우저는 줄바꿈을 무시하기 때문에 웹에서 보았을 때의 시각적인 분리 효과가 전혀 없음 <br />

1. <ol> - </ol> : ordered list의 약자로 순서 있는 목록을 지칭한다. <br />
-> <ol type="" start="">와 같이 ol 태그 내부에 타입과 시작 문자를 지정할 수 있다. <br />
!! type : 순서 목록의 숫자 조정 (1: 숫자, a: 소문자, A: 대문자, i: 로마소문자, I: 로마대문자) <br />
!! start : 목록의 시작 번호 수정 (숫자로 입력) <br />

2. <ul> - </ul> : unordered list의 약자로 순서 없는 목록을 지칭한다. <br />
-> 이후 css를 통해 불릿(문서의 앞부분 동그란것)의 모양을 바꿀 수 있다. <br />

3. <dl> - </dl> : description list의 약자로 설명 목록을 지칭한다. <br />
-> 평소 우리가 필기를 할때 한 주제에 대한 정의나 설명을 콜론(:)으로 구분하지만, 웹 사이트의 경우 그러한 규칙이 없기 때문에 앞으로 나올 요소들은 설명을 위한 요소들임을 알려주는 태그 <br />
-> 일반적인 ol과 ul 태그와 다르게 내부에 <li>태그를 사용하는 것이 아닌 <dt>와 <dd>라는 새로운 태그를 사용한다. <br />
!! <dt> - </dt> : 설명에 대한 주체로 고양이의 대한 설명을 한다고 가정했을때 '고양이' 라고 작성 <br />
!! <dd> - </dd> : dt에 대한 설명 리스트로 dt인 '고양이'에 대한 특성을 나열 <br />
