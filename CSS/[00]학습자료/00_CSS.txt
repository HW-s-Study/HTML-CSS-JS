### Cascading Style Sheet
Cascading(우선순위가 위에서 아래로 적용되며 스타일끼리 충돌하지 않도록 막아주는 역할)이 적용된 우선순위가 있는 스타일 시트로 문서의 겉모습을 결정하는 디자인(HTML에서 자주 사용하는 글꼴이나 색상, 정렬, 각 요소의 배치 방법 등)을 담당 <br />
웹 요소에 둘 이상의 스타일을 적용할 때 우선순위에 따라 적용할 스타일을 결정 <br />
-> 기존 HTML에서 <font> 태그를 이용해 사용하던 태그를 '정보'와 '디자인'으로 분리하면서 웹 페이지의 역할을 분리 <br />
!! 이전에는 font태그의 개수만큼 스타일을 변경해야 했다면, 중복을 제거해 태그의 개수가 몇 개든 한번에 변경할 수 있도록 해 유지보수를 편리하게 하고 가독성을 높힘 <br />

### CSS의 기본 구조
```
선택자(Selecter) {
  속성(Property):값(Value);      // 선언, 효과(Decolaration)
}
```
!! 여기서 세미콜론(;)은 구분자로서 효과를 구분하는 용도로 사용


## 스타일 시트의 종류와 구분(CSS1 학습지 첫장 참고)
- 브라우저 기본 스타일
- 사용자 스타일(인라인 스타일, 내부 스타일 시트, 외부 스타일 시트)


### 브라우저 기본 스타일
: 웹 브라우저에 기본적으로 만들어져 있는 스타일(<h1> ~ <h6>과 같은 태그의 스타일을 의미)

### 사용자 스타일
: 사이트 제작자, 개발자가 직접 만들어 부여한 스타일

- 인라인 스타일 : 스타일 적용 대상에 직접 표시하는 스타일
ex. ```<h2 style="color:blue;">레드향 샐러드 레시피</h2>```

- 내부 스타일 시트 : 웹 문서(HTML) 안에 사용할 스타일을 같은 문서 안에 정리
ex. ```
<style>
  p {
    text-align: right;
    color: brown;
  }
</style>
```
#### <style> 태그
HTML의 문법이면서 동시에 CSS이므로 CSS문법에 맞게 처리해야 한다는 의미로 HTML 파일 내부에 존재하는 '내부 스타일 시트' <br />
둘은 서로 완전히 다른 언어이므로 어디부터 어디까지가 CSS 문법인지 설명해 주어야 하기 때문에 그 역할을 담당 <br />
!! HTML 태그 내부에 있는 style=""는 HTML의 속성, style이라는 속성은 값으로 반드시 CSS 효과가 들어온다고 약속되어 있는것일 뿐임으로 <style>태그와 다름 <br />

- 외부 스타일 시트 : 스타일 정보를 따로 저장, .css 파일을 생성하여 스타일을 정리 한 후 링크 태그를 이용해 스타일 적용
ex. ```<link rel="stylesheet" href="파일명.css">```


### CSS 선택자와 우선순위
- !importent : 어떤 스타일보다 우선 적용( 속성:값 !importent; )
- 인라인 스타일 : 태그 안에 style 속성을 사용해 해당 태그만 스타일 적용

- id 선택자 : 특정 부분에 스타일을 한 번만 적용( #아이디명{속성:값;} )
- 클래스 선택자 : 특정 부분에 재사용 가능한 스타일 적용( .클래스명{속성:값;} )
- 타입 선택자 : 특정 태그를 사용한 모든 요소에 스타일 적용( 태그명{속성:값;} )
- 전체 선택자 : 스타일을 문서의 모든 요소에 적용할 때 사용( *{속성:값;} )

- 그룹 선택자 : 같은 스타일 규칙을 사용하는 요소들을 그룹화하는 역할로 각각의 가중치에 따라 우선순위를 부여( 선택자1, 선택자2{속성:값;} )
!! 우선순위와 적용 범위가 같다면 소스 코드 순서로 우선순위가 정해짐(나중에 작성한 스타일이 먼저 작성한 스타일의 우선순위가 더 높음)

#### 스타일 상속
포함하는 태그: 부모요소
포함되는 태그: 자식요소
!! 별도 스타일을 지정하지 않으면 부모요소의 스타일이 자식요소로 상속
